## 2. JSON 数据契约

### 2.1 图谱

```json
{
  "schema_version": "...",
  "schema": { "node_types": [], "relation_types": [] },
  "nodes": [
    {
      "id": "course_java__curriculum_...",
      "name": "知识点名称",
      "type": "KnowledgePoint",
      "description": "说明",
      "aliases": [],
      "confidence": 0.98,
      "source_files": [],
      "source_pages": []
    }
  ],
  "edges": [
    {
      "id": "edge_...",
      "source": "源节点ID",
      "target": "目标节点ID",
      "type": "part_of",
      "evidence": "关系证据"
    }
  ],
  "metadata": { "version": "v2026.07.18" }
}
```

前端应以 `id` 作为稳定主键、以 `name` 作为展示名。不得通过展示名反向猜测节点 ID。`part_of` 的方向为“子节点 -> 父节点”。

### 2.2 题库与映射

题库主键为 `question_id`，映射文件使用相同 `question_id` 关联；每个 `links[]` 元素中的 `knowledge_node_id` 必须存在于同版本正式图谱。

```json
{
  "question_id": "XDU_...",
  "stem": "题干",
  "answer": "答案",
  "answer_status": "source_verified_complete",
  "course_id": "course_cpp"
}
```

```json
{
  "question_id": "XDU_...",
  "links": [
    {
      "knowledge_node_id": "course_cpp__...",
      "knowledge_name": "知识点名称",
      "role": "primary",
      "mapping_status": "approved",
      "evidence": "直接考查证据"
    }
  ]
}
```

## 3. API 契约

前端配置：

```text
VITE_KG_API_BASE=http://127.0.0.1:8000
```

| 方法 | 接口 | 用途 |
| --- | --- | --- |
| GET | `/api/health` | 检查 API 已加载的图谱、题库、映射数量 |
| GET | `/api/graphs/root` | 获取根图 |
| GET | `/api/graphs/{nodeId}?depth=1` | 获取某节点的前端子图 |
| GET | `/api/nodes/{nodeId}` | 获取节点详情 |
| GET | `/api/nodes/{nodeId}/neighbors?depth=1` | 获取邻居子图 |
| GET | `/api/search?q=关键词&limit=8` | 搜索知识点或题目 |
| GET | `/api/questions` | 获取题目列表 |
| GET | `/api/questions/{questionId}` | 获取题目详情 |
| GET | `/api/schema` | 获取 Schema、元数据和类型列表 |

接口错误应以 HTTP 状态码和 JSON `error` 字段返回。前端 API 地址为空时会使用模拟数据；正式验收必须配置为本地 API。

## 4. 前端视图数据适配

前端 `graphApi.js` 使用 `GET /api/graphs/:graphId`、`GET /api/nodes/:nodeId`、`GET /api/search`。API 返回的前端节点至少包含：

```text
id, label, type/sourceType, summary, confidence,
aliases, neighbors, questions, children, relationCount, sourceCount
```

`children` 存在时表示可继续下钻。图谱边使用 `source`、`target`、`label`、`type`、`strength`。前端不要依赖模拟数据中的固定课程或固定节点 ID。

## 5. 更新规则

1. 后端先生成新的发布目录和 `release_manifest.json`。
2. 验证节点、关系、题库和映射的一致性后，原子替换前端副本。
3. 前端通过 API `/api/health` 或发布清单核对版本与数量。
4. Neo4j 执行完整替换导入后，再以实际查询统计核验。
5. 不允许单独修改前端 JSON 而不更新发布版、题库映射或哈希。
