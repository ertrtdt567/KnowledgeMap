# 编程领域知识图谱模块系统架构设计

## 1. 架构目标

架构将“资料中的非结构化教学内容”转换为“可查询、可视化、可映射习题的正式知识图谱”，并保证每个阶段可追溯、可审计、可替换。

```text
课程资料（PPT / PDF / Word / 习题）
            |
            v
资料预处理 -> 统一知识片段 clean_chunks.json
            |
            v
规则 + 大模型抽取 -> 候选实体与关系
            |
            v
规范化、融合消歧、Schema 校验 -> 标准图谱
            |
            +--> 课程知识树、跨课程核心概念映射
            |
            +--> 题目解析、答案核验、题目—知识点映射
            v
正式发布 v2026.07.18
   |                |                |
   v                v                v
前端 JSON       RDF/TTL/XML       Neo4j/Cypher
   |                                  |
   +-------- 前端 API / React ----------+
```

## 2. 逻辑分层

### 2.1 数据输入层

- 课程课件与教材：PPT、PDF、Word。
- 带答案题目：题干和答案按题号匹配后入正式题库。
- 无答案或证据不足题目：保留在候选/审核队列，不直接进入正式题库。

### 2.2 构建层

- `preprocess_materials.py`：提取文本、去噪、切分、保存文件和页码来源。
- `extract_graph.py`：使用确定性规则和大模型抽取实体、关系、代码示例等候选数据。
- `normalize_graph.py`：实体归并、别名处理、关系合法性校验、证据和置信度整合。
- `build_course_centered_graph.py`：按课程挂载本地知识树，并以 `maps_to_core` 连接经过证据验证的跨课程公共概念。
- `audit_formal_graph.py`、关系与题目复核脚本：阻止结构性错误进入发布版。

### 2.3 模型层

主层级遵循：

```text
编程领域
  -> 课程
     -> 知识领域
        -> 知识单元
           -> 知识点
              -> 代码示例 / 语法元素 / 代码结构
```

- `part_of`：课程内部层级归属关系。
- `maps_to_core`：课程局部概念映射到全局核心概念，不替代课程树。
- `prerequisite_of`：学习先后依赖。
- `has_code_example`、`appears_in_example`、`syntax_used_in_example`：知识点与代码资源关联。
- `assesses`：习题指向其直接考查的知识点。

### 2.4 存储与服务层

- JSON：正式数据源，也是前端离线/静态加载源。
- RDF：用于语义数据交付和 RDF 解析验证。
- Neo4j：用于图查询、可视化与 Cypher 演示。
- 轻量 API：使用 Python 标准库读取正式 JSON 并提供前端视图数据；不依赖 Neo4j 才能展示基本图谱。

### 2.5 展示层

仓库前端位于 `kg-web-frontend/`，技术栈为 React、Vite、Cytoscape.js、Lucide。当前实现支持课程图谱、子图下钻、节点详情、节点搜索、代码示例原题和习题答案查看。

## 3. 核心设计约束

1. 课程局部节点不因名称相似而直接合并；只有有充分课程证据的概念才建立全局核心映射。
2. `aliases` 仅放同义词、缩写或译名；可独立学习的概念应作为节点和关系展示。
3. `CodeExample` 作为叶子资源，不允许以知识树父节点身份扩张。
4. 题目映射仅保留题干、代码或答案能直接证明的知识点；背景相关关系不作为正式 `assesses`。
5. 所有发布数据使用唯一版本和哈希；前端、API、RDF、Neo4j 不得混用不同版本。

## 4. 当前接口实现边界

前端默认可使用模拟数据；当配置 `VITE_KG_API_BASE` 后，会调用 API 的 `/api/graphs`、`/api/nodes`、`/api/search` 和题目接口。API 启动时必须通过参数显式传入本发布版的图谱、题库和映射路径，避免其历史默认路径读取旧数据。
