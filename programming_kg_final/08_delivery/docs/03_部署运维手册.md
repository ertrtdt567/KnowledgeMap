# 编程领域知识图谱模块部署运维手册

## 1. 适用范围

本手册适用于 Windows 本机演示环境，不代表已部署到公网或生产服务器。演示由三个独立进程组成：Neo4j Desktop 数据库、Python 图谱 API、React/Vite 前端。

## 2. 环境要求

| 组件 | 要求 | 用途 |
| --- | --- | --- |
| Windows 10/11 | 本机用户可写目录 | 演示环境 |
| Python 3.10+ | 建议使用项目统一 Python | API、导入、审计脚本 |
| Node.js 18+ 与 pnpm | 前端依赖安装与启动 | React/Vite 前端 |
| Neo4j Desktop | 本地实例，Bolt 默认端口 7687 | 图数据库展示与查询 |
| 可选：DashScope API Key | 仅重新构建图谱时需要 | 大模型抽取、候选复核 |

## 3. 正式版本与数据位置

设仓库根目录为 `$REPO`，正式发布目录为：

```powershell
$RELEASE = "$REPO\work\oop_kg_demo\output\programming_kg\release\v2026.07.18"
```

必须使用下列三项同版本文件：

```text
$RELEASE\standard_graph_v2026.07.18.json
$RELEASE\questions_v2026.07.18.json
$RELEASE\question_knowledge_links_v2026.07.18.json
```

不要把历史 `standard_graph.json`、早期 OOP Demo 输出或候选图谱作为演示数据源。

## 4. Neo4j 启动与正式入库

1. 在 Neo4j Desktop 启动项目实例，确认 `neo4j` 数据库为 `RUNNING`，Bolt 地址为 `bolt://127.0.0.1:7687`。
2. 在 PowerShell 设置密码。该变量只在当前终端有效：

```powershell
$env:NEO4J_PASSWORD = "你的Neo4j密码"
```

3. 在仓库根目录运行正式覆盖导入：

```powershell
python work\oop_kg_demo\import_to_neo4j.py `
  --input "$RELEASE\standard_graph_v2026.07.18.json" `
  --output-dir "$RELEASE\neo4j_import" `
  --execute --clear `
  --uri bolt://127.0.0.1:7687 `
  --database neo4j
```

`--clear` 会清除当前 `neo4j` 数据库中的全部图节点后重建。因此只应对本项目专用数据库执行；若需保留旧数据，应先在 Neo4j Desktop 备份或创建独立数据库。

4. 导入成功后，在 Neo4j Browser 运行：

```cypher
MATCH (n) RETURN count(n) AS nodes;
MATCH ()-[r]->() RETURN count(r) AS edges;
MATCH (q:Question)-[r]->() WHERE r.type = 'assesses' RETURN count(r) AS mappings;
```

预期分别为 `2612`、`6059`、`73`。

## 5. 启动 Python 图谱 API

仓库中前端配套 API 位于：

```text
$REPO\work\KnowledgeMap_publish_clone\oop_kg_demo\api_server.py
```

在独立 PowerShell 窗口执行：

```powershell
python "$REPO\work\KnowledgeMap_publish_clone\oop_kg_demo\api_server.py" `
  --host 127.0.0.1 --port 8000 `
  --graph "$RELEASE\standard_graph_v2026.07.18.json" `
  --questions "$RELEASE\questions_v2026.07.18.json" `
  --question-links "$RELEASE\question_knowledge_links_v2026.07.18.json"
```

浏览器访问 `http://127.0.0.1:8000/api/health`，应返回图谱、题库和映射统计。该 API 读取 JSON，不要求 Neo4j 已启动；Neo4j 用于数据库演示和 Cypher 查询。

## 6. 启动前端

1. 进入前端目录：

```powershell
Set-Location "$REPO\work\KnowledgeMap_publish_clone\kg-web-frontend"
```

2. 创建 `.env` 并设置 API 地址：

```powershell
Copy-Item .env.example .env
Set-Content .env "VITE_KG_API_BASE=http://127.0.0.1:8000"
```

3. 安装并启动：

```powershell
corepack pnpm install
corepack pnpm dev
```

按终端提示访问 Vite 地址，通常为 `http://localhost:5173`。若 `.env` 中未设置 `VITE_KG_API_BASE`，前端会回退到模拟数据，不能作为正式验收演示。

## 7. 日常运维与故障处理

| 现象 | 排查与处理 |
| --- | --- |
| 前端显示模拟图谱 | 检查 `.env` 的 `VITE_KG_API_BASE`，修改后重启 Vite |
| 前端请求失败 | 访问 `/api/health`，检查 API 终端是否仍在运行、端口是否为 8000 |
| API 加载旧数据 | 检查启动命令是否显式传入三个 `v2026.07.18` 文件 |
| Neo4j 导入冲突 | 使用最新版 `import_to_neo4j.py` 并确认项目数据库可安全执行 `--clear` |
| Neo4j 连接失败 | 检查实例状态、Bolt 端口 7687 和 `NEO4J_PASSWORD` |
| 图谱构建时模型失败 | 保留已有缓存和候选输出；检查 `DASHSCOPE_API_KEY`、模型额度与网络，不将失败候选直接入正式图谱 |

## 8. 备份与恢复

- 最小备份：保存整个 `v2026.07.18` 发布目录，特别是 `release_manifest.json` 和三份正式 JSON。
- Neo4j 变更前：使用 Neo4j Desktop 的数据库备份能力，或在独立项目数据库中操作。
- 恢复正式图谱：重新执行第 4 节导入命令；正式 JSON 是恢复的唯一数据源。
- 不应备份或提交 API 密钥、Neo4j 密码和大模型响应缓存。
