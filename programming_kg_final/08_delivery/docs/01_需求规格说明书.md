# 编程领域知识图谱模块需求规格说明书

## 1. 模块定位

本模块为智能编程辅导系统提供可追溯的编程知识组织、检索和习题关联能力。它不替代在线判题、用户画像或教学推荐子系统，而是为这些能力提供统一的课程知识、代码示例和习题知识点依据。

当前正式版本覆盖 Java、Python、C++、数据结构、UML 建模设计与分析五门课程；通过全局核心概念连接跨课程共性知识，同时保留各课程内部的知识层级和来源证据。

## 2. 功能性需求

| 编号 | 需求 | 实现结果 |
| --- | --- | --- |
| FR-01 | 接收 PPT、PDF、Word 等课程资料并形成统一知识片段 | 已实现资料解析、清洗、切分和来源页码保留 |
| FR-02 | 从知识片段抽取实体、关系与代码结构 | 已实现规则与大模型协同抽取，保留抽取证据与置信度 |
| FR-03 | 规范化同义实体、关系名称和节点类型 | 已实现 Schema 约束、别名清洗、融合消歧与无效关系过滤 |
| FR-04 | 构建课程中心的层级知识图谱 | 已实现“编程领域 → 课程 → 知识领域 → 知识单元 → 知识点”主层级；最大层级深度为 7 |
| FR-05 | 表达跨课程公共概念 | 已实现课程局部知识点到全局核心概念的 `maps_to_core` 映射，不以名称相同就强行融合 |
| FR-06 | 管理代码示例 | 已实现 `CodeExample`、代码结构、语法元素与知识点之间的受约束关系；代码示例作为叶子类资源使用 |
| FR-07 | 管理习题、答案和题目映射 | 已实现正式题库、来源答案核验、题目类型/难度/能力属性与 `assesses` 知识点映射 |
| FR-08 | 为前端提供课程图、子图、节点详情和搜索数据 | 已实现 JSON 数据交付与轻量 API 的图谱、节点、搜索、题目接口 |
| FR-09 | 导出并存储图谱 | 已生成 JSON、RDF/Turtle、RDF/XML、Neo4j Cypher；正式数据已导入 Neo4j |
| FR-10 | 输出可审计质量结果 | 已提供结构审计、关系抽样复核、课程知识点复核、题目映射全量复核和版本清单 |

## 3. 非功能性需求

| 维度 | 要求 | 当前实现 |
| --- | --- | --- |
| 正确性 | 图中不应出现悬空边、自环、失效题目映射 | 正式审计通过；无悬空边、自环和失效题目映射 |
| 可追溯性 | 节点与关系应可定位到课程资料或题目来源 | 节点保留 `source_files`、`source_pages`、证据片段等属性 |
| 可维护性 | 支持用新资料增量构建，并能区分候选与正式数据 | 输入、抽取、规范化、课程树、题目、发布阶段均有独立脚本与产物目录 |
| 一致性 | JSON、前端、RDF、Neo4j 版本一致 | 由发布清单、SHA256 和跨格式核验约束 |
| 可解释性 | 前端可以展示节点定义、别名、来源、关联节点和练习题 | API 提供节点详情、邻居、题目与映射数据 |
| 安全性 | 密钥和密码不得进入代码仓库 | `DASHSCOPE_API_KEY`、`NEO4J_PASSWORD` 仅通过环境变量传入 |
| 可用性 | 支持 Windows 本机演示 | 已按本机 Python、Neo4j Desktop、React/Vite 前端给出部署流程 |

## 4. 数据范围与边界

- 正式图谱：2,612 个节点、6,059 条关系。
- 课程数量：5 门。
- 正式题库：45 道题，73 条题目—知识点映射。
- 本模块不承诺对所有可能的潜在知识点做穷尽召回；只将证据充分、通过约束和复核的节点、关系、题目纳入正式图谱。
- 大模型用于资料抽取、候选关系判别和辅助复核；正式发布前必须经过 Schema、课程树与质量审计约束，不能将模型原始输出直接作为正式图谱。
