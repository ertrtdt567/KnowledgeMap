# 编程领域知识图谱升级方案与对比

## 1. 升级目标

旧版以“面向对象编程”为示例，已经完成了从课程材料到 Neo4j 的 Demo 闭环。本次升级不删除旧版，而是在其基础上构建面向完整编程领域的正式流程：课程知识结构由人工可维护的标准目录约束，Java、Python、C++ 等只是该目录中的语言实现与证据来源。

旧版产物已保存于 `output/archive/oop_demo_20260711`，原 Neo4j 数据未被本次离线验证覆盖。

## 2. 旧版与改进版对比

| 对比项 | 旧 OOP Demo | 改进后的编程领域版本 |
| --- | --- | --- |
| 建设范围 | 以面向对象知识为中心 | 程序基础、控制、函数与模块、数据组织、OOP、异常、文件、工程实践、并发线程九个领域 |
| 知识层级来源 | 抽取后再规范化，课程层级约束较弱 | `data/programming_curriculum_v0_2.json` 是唯一权威目录；模型不能自由定义教学层级 |
| 原始课程片段 | 476 个，经过 OOP 关键词过滤 | 878 个，覆盖现有 Java 与 Python 课程材料 |
| 实体设计 | OOPConcept 等偏示例化类型 | 课程层 `KnowledgeDomain/KnowledgeUnit/KnowledgePoint` 与技术层 `CodeExample/CodeStructure/SyntaxRule/ProgrammingLanguage` 分离 |
| 代码示例位置 | CodeExample 可作为关系起点 | CodeExample 强制为叶子节点，只能被知识点、语法规则或代码结构指向；标准图谱已校验 281 个代码示例的出边为 0 |
| 关系处理 | 有关系规范化，但自动层级关系可能较宽泛 | 课程层级仅由标准目录生成；自动发现但目录外的知识点进入候选报告，不进入正式图谱 |
| API 使用 | 选择模型时可对所有片段调用 | 默认规则优先；当前材料中预计 821 个片段直接走规则，仅 57 个低置信度或无语义实体片段需要模型辅助 |
| 正式图谱结果 | Neo4j 已导入 1003 节点、3126 关系（Demo 结果） | 离线验证得到 588 个正式节点、2004 条正式关系；规模变小是因为只保留有课程证据且可归入标准目录的内容 |
| 层级质量 | 未将“唯一父节点、无环、证据覆盖”作为独立准入条件 | 47 条课程层级边；无非法父节点、无环、直接覆盖节点均有材料证据，目录外候选为 0 |
| 习题支持 | 20 道样例题，99 条“考察”关系 | 可解析 Word 习题；已从 C++ 练习文档得到 25 道可处理习题，支持答案来源、生成、独立复核、正式题库准入和审核清单 |
| 答案可信度 | 样例题自带答案 | 教师提供答案可直接入库；模型答案须二次复核且置信度不低于 0.85，未通过者绝不进入正式题库 |

## 3. 改进版总体流程

```text
课程 PPT/PDF/Word
  -> 清洗、去重、切片、语言与领域标注
  -> 规则抽取（语言、语法、代码结构、显式知识点）
  -> 仅对低置信度/无语义实体片段调用模型
  -> 实体关系规范化
  -> 标准课程目录对齐、融合、消歧和层级校验
  -> 正式标准图谱 standard_graph.json
  -> Neo4j 属性图与中文展示关系

带答案习题 -> 直接标准化 -> 正式题库
无答案习题 -> 模型生成参考解答 -> 独立复核 -> 达阈值后进入正式题库
                                                   -> 未达阈值进入人工审核清单
正式题库 -> 习题-知识点映射 -> Neo4j Question/考察 关系
```

## 4. 当前已经验证的结果

### 4.1 课程知识图谱主链

- 成功处理 878 个课程知识片段。
- 规则抽取结果：647 个去重实体、2330 条关系；平均实体置信度 0.9235，平均关系置信度 0.8773。
- 规范化后：640 个节点、2322 条关系。
- 标准目录增强后：588 个正式节点、2004 条正式关系。
- 层级校验通过：47 条层级边、0 个非法父节点、0 个环、0 个无证据的直接覆盖知识节点、0 个目录外候选知识点。
- 已成功生成 Neo4j 的 Cypher 导入脚本和展示查询；本轮未执行 `--execute`，因此不会覆盖现有 OOP 图数据库。

### 4.2 习题链

- C++ Word 练习文档解析得到 25 道可处理习题，包括类与对象、抽象类、继承、多态、队列、栈、指针与引用、函数重载、类型转换、模板和代码改错等内容。
- 文档前言、答案区以及只引用外部 PDF 但没有实际题干的文字，已被排除，不会误入题库。
- 离线准入测试中，25 道缺答案习题全部进入审核清单，正式题库为 0。这是预期的保护行为，不是失败。

## 7. 结论

改进版不再只是“把 OOP 术语抽出来”的 Demo，而是受标准课程树约束的编程领域知识图谱。它的核心质量保证来自三道关：课程目录约束、证据可追溯与层级校验；习题层额外增加答案来源和复核准入。当前主图谱已完成并离线验证，习题正式入库只差在可联网环境运行模型生成与复核步骤。
